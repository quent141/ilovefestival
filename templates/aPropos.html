À PROPOS: Documentation technique de la page ILoveFestivals

Introduction

Notre site web permettra de répertorier des festivals. Ceci permet de faciliter la recherche de festivals existants en France et à l’International pour des festivaliers intéressés par des artistes ou des genres musicaux en particulier.

“Mais où joue *insérer nom d’artiste* cet été ?” C’est une question que beaucoup de festivaliers ont dû se poser à l’heure de choisir leur(s) festivals de l’été. Objectif: Créer un site web de recherche de festivals selon le genre, les artistes, la popularité etc. Aujourd’hui, il n’existe pas de comparateurs/répertoires de festivals répondant de manière simple à la question précédente ! Les futurs festivaliers se retrouvent obligés d’éplucher les sites des festivals ou des artistes pour trouver des dates leur convenant. Notre projet faciliterait donc la recherche en mettant à disposition un outil intuitif et facile d’utilisation, avec entre autres un système de recherche par artiste ou par genre, ainsi qu’une partie réservée aux avis et aux commentaires sur les festivals et les artistes. Cela serait un point fort pour notre site puisque les festivals et les artistes seraient notés par nos utilisateurs. Cette action permettrait de créer un centre de partage entre les festivals, les artistes et les festivaliers afin de permettre aux festivals et aux artistes de s’améliorer pour la prochaine fois s’ils ont été mal notés, et cela permettrait aux festivaliers d’avoir un avis “concret” sur un festival ou un artiste.

Conception de la base de données

La base de données SQL a été conçue à l’aide du soft phpmyAdmin, qui permet à la fois d’avoir une interface graphique intuitive et de modifier facilement et rapidement la base. Au vu des nombreuses relations entre les tables que notre base induisait, nous avons fait le choix d’avoir recours à plusieurs tables de passage. Elles facilitent la structure des requêtes puisque que de simples jointures permettent de faire des requêtes sur plusieurs tables simultanément.

Améliorations possibles :

  • - La base de données n’est pour l’instant remplie qu’avec un échantillon suffisant d’artistes et de festivals pour faire une démonstration. Bien sûr, le site gagne en intérêt quand la base est bien remplie.
  • - Justement, on pourrait mettre au point un bot de remplissage pour la base, puisque la remplir à la main prendrait beaucoup de temps vu la quantité d’infos à traiter.
  • - Intégration d’un module de géolocalisation type Google Maps pour ajouter une dimension proximité du festival.
  • - Version multilingue.

Maquette du site / Front-End

Chaque page du site possède un menu latéral permettant d’accéder aux autres pages et revenir à l’accueil.Toutes les pages ont aussi une option de connexion/déconnexion pour l’utilisateur.

Notre site se divise en 10 pages :

  • - La page d’accueil :Elle affiche quelques nouvelles sur les différents festivals du moment
  • - La page de login :Elle permet de s’enregistrer dans le site web ou de connecter si un compte a déjà été créé. Le mot de passe et le login sont enregistrés dans la base de données.
  • - Pages de classements (3) :Permet l’affichage des classements des trois dernières années sous forme de tableau.
  • - Page artistes :Permet d’afficher et rechercher les différents artistes avec leur photo, leur note et les prochains festivals auxquels il participera.
  • - Page festivals :Permet d’afficher et rechercher les différents festivals de la base de données avec une photo, leur note, les prix (Allant de 0 à 4 du moins au plus cher) et le lien vers la page officielle du festival.
  • - Page découverte :La page découverte affiche dans un “slider” les principaux festivals du moment . Chaque “slide” a un lien vers la page officielle du festival. Le titre ” top10” a un lien vers une page contenant l’ensemble de festivals du moment. Ensuite il y a deux blocs (hot et recommandations). Chaque bloc a un lien vers la page correspondante.
  • - Page Hot: Page qui affiche l’ensemble des festivals les plus connus. Cette page contient une barre de filtrage pour retrouver les festivals avec les liens vers leur page correspondante.
  • Page recommendation : Page qui affiche l’ensemble des festivals recommandés pour l’utilisateur selon les intérêts qu’il a enregistrés.
  • - Page profil utilisateur : Page personnelle avec une option de logout, l’option de saisir une description personnelle et ajouter des gouts personnels (artistes, genres, …).
  • - Page à propos : Page avec la documentation technique sur le site, la description des pages et quelques subtilités sur le site. EIle possède aussi une description de la base de données pour mieux comprendre l’architecture interne du site.

Note: Lorsque l’utilisateur est connecté le logo du menu latéral possède un lien vers la page de profil . Dans le cas contraire le lien est vers la page d’accueil du site.

Choix technologiques

D’une part, nous avons utilisé python/Flask comme technologie pour relier la base des données avec le Front-End parce que nous avons été introduit à ces outils pendant les cours de WEB.

Flask

Certes, flask ne possède pas des outils de bootstrapping mais vu que nous ne cherchons pas à avoir une application MVC de grande taille, ceci ne pose pas de problème.

Python

L’avantage de python c’est qu’il met à disposition de bonnes librairies standard et de bons frameworks. Ceci permet de developper plus rapidement et le code est plus facile à lire et à écrire. Nous avons aussi utilisé python parce que c’est un langage que nous n’utilisons pas souvent et qui est pourtant intéressant.
